## 🛠 DESIGN SYSTEM v1.4 (LOCKED)

> Geëxtraheerd uit `06-landing-page/public/index.html` (v3 implementatie).  
> Alle tokens, type-scale en patterns zijn 1:1 hergebruikbaar in landing, PDF, OG-images, social-cards en pitch-decks.

### 1. Design tokens (CSS-vars)

```css
:root {
  /* Brand */
  --orange:        #FF6B1A;     /* primary accent / CTA */
  --orange-shade:  #E55A0F;     /* hover / pressed */

  /* Authority blues */
  --blue:          #1E3A5F;     /* headers, body-titles, borders */
  --blue-deep:     #15293F;     /* dark blocks (stack, modules) */
  --blue-pale:     #2C5582;     /* hover state op modules */

  /* Tech accent */
  --cyan:          #00D4FF;     /* AI / data accent — sparingly */
  --cyan-mute:     #4DBFD9;     /* secondary cyan */

  /* Neutrals */
  --warm-grey:     #F5F0EB;     /* page background, body */
  --warm-grey-2:   #EAE3D9;     /* alt background, author block */
  --ink:           #1A1A1A;     /* body text */
  --ink-soft:      #2D2D2D;     /* secondary text */

  /* Lines */
  --rule:          rgba(30, 58, 95, 0.18);   /* hairline borders */

  /* Spacing baseline */
  --grid:          8px;          /* alle padding/margin = veelvouden van 8 */
}
```

**Telling:** 13 design-tokens (10 kleur + 1 rule + 1 grid + 1 base scale).

**Gebruik:** ALTIJD via CSS-vars (`var(--orange)`), nooit hardcoded hex. Voor PDF/print: dezelfde hex-waarden in WeasyPrint / InDesign.

### 2. Type-scale

| Token | Font · Size · Weight | Line-height | Letter-spacing | Gebruik |
|-------|----------------------|-------------|----------------|---------|
| **Display H1** | Inter 900 · `clamp(48px, 8vw, 104px)` | 0.9 | -0.04em | Hero, manifest |
| **Display H2** | Inter 800-900 · `clamp(40px, 5.5vw, 72px)` | 0.95 | -0.03em | Stack-header, case-title |
| **Subhead** | Inter 800 · 26px | 1.1 | -0.01em | Module h3, card-titles |
| **Standfirst** | Inter 400 · `clamp(18px, 2vw, 22px)` | 1.5 | — | Hero-lede, intro |
| **Body** | Inter 400 · 18px | 1.65 | — | Essay, body-paragrafen |
| **Mono labels** | JetBrains Mono · 10-11px UPPERCASE | 1.3 | 1.5-2px | `[KEY]`, annotations, corner-labels |
| **Mono ID-codes** | JetBrains Mono 700 · 9-13px | 1.3 | 1-2px | DOC-ID, M01, T = 0 |

**Regel:** Inter voor alle prozaische tekst. JetBrains Mono ALLEEN voor labels, IDs, terminal-output. Geen serif ANYWHERE.

### 3. Pattern library (reusable design-blocks)

#### A. Blueprint grid (background)

Vaste 64px × 64px grid achter heel de pagina, met radial-mask zodat het centrum scherp is en de randen vervagen.

```
+----+----+----+----+
|    |    |    |    |   <- 64px × 64px grid lines
+----+----+----+----+      kleur: var(--rule)
|    | ☉  |    |    |      mask: radial-gradient
+----+----+----+----+      opacity: 0.35
|    |    |    |    |
+----+----+----+----+
```

#### B. Spec-sheet patroon

Bordered table met `[KEY]` mono labels (10px, UPPERCASE, 1.5px letter-spacing) + 14px Inter values.

```
┌────────────────────────────────────────┐
│ [SPEC-SHEET — ING. W. ARTS]      ●   │  <- header, blue bg, mono uppercase
├────────────┬───────────────────────────┤
│ NAAM       │ Wouter Arts               │
├────────────┼───────────────────────────┤
│ TITEL      │ Ing. (Werktuigbouwkunde)  │
├────────────┼───────────────────────────┤
│ ROL        │ Recruitment Engineer      │
└────────────┴───────────────────────────┘
```

#### C. Terminal CLI block

Donker (var(--blue-deep)) blok met `$ command` regel + cyan response-line. Mono-only.

```
┌─────────────────────────────────────────┐
│ $ subscribe --stack=2026                │  <- prompt orange "$"
│ > sending PDF to inbox...               │  <- response cyan
│                                         │
│  > ____________________________         │  <- input field, cyan border-bottom
│                                         │
│  [DOWNLOAD PDF →]                       │  <- orange CTA, mono uppercase
└─────────────────────────────────────────┘
```

#### D. Corner-labels

Top-right mono micro-text (9px, cyan/orange, 1-1.5px letter-spacing) op cards.

```
┌──────────────────────┐
│                M01   │  <- corner-label top-right
│                      │
│   Spec content       │
│                      │
└──────────────────────┘
```

Voorbeelden: `M01`, `∆ INPUT`, `T = 0`, `CLASSIFICATIE: PUBLIEK`.

#### E. § markers (section annotations)

`§ 01 · MANIFEST` — orange of cyan annotation met 2px border-top, mono 11px UPPERCASE 2px letter-spacing.

```
━━━━━━━━━━━━━━━━━━           <- 2px border-top, var(--orange)
§ 01 · MANIFEST              <- mono uppercase, var(--orange)
```

Cyan variant op donkere achtergrond (specs-bar, stack-section).

#### F. Module-IDs

`M01 · Sourcing` — mono format met orange num + grey name.

```
M01 · SOURCING               <- num=orange weight 700, name=grey 50% opacity
```

Hover: num verandert naar cyan, achtergrond shift naar `--blue-pale`.

#### G. Doc-id badges

Orange filled badge + DOC-ID code + CLASSIFICATIE: PUBLIEK.

```
[DOC-2026-001]  ─────────────  CLASSIFICATIE: PUBLIEK
 ▲                ▲                ▲
 orange filled    1px hairline     mono uppercase 11px
 mono 700         max-width 200px  letter-spacing 2px
```

#### H. Byline-block

Bordered grid (1px solid `--blue`) van label/value pairs, met vertical dividers tussen blokken.

```
┌─────────────┬──────────────┬──────────────┐
│ AUTEUR      │ DATUM        │ EDITIE       │  <- label: mono 10px 1.5px
│ Ing. W.Arts │ MEI 2026     │ V3           │  <- value: Inter 14px 600
└─────────────┴──────────────┴──────────────┘
```

### 4. Animation rules

| Animatie | Trigger | Specs |
|----------|---------|-------|
| **fadeUp** | Hero-elementen op load | `opacity 0→1 + translateY(8px→0)`, 0.5s ease-out, staggered 0.05s/0.1s/0.2s/0.3s |
| **blink-pulse** | Status-dots in headers | `opacity 1→0.3` op 95% van 2s loop |
| **hover-shift** | Module-cards | `background var(--blue-deep) → var(--blue-pale)` + `num color → var(--cyan)`, 0.3s ease |
| **prefers-reduced-motion** | OS-instelling | Alle animations + transitions naar 0.01ms (effectief uit) |

**Regel:** Geen scroll-jank, geen parallax, geen "designer-flexen". Subtle, doelgericht, functioneel.

### 5. Voice rules (uitgebreid v1.4)

Re-affirm vanaf v1.3 + nieuwe regels:

1. **Trust-marker volgorde** — Beutech mag NOOIT 2× achter elkaar voorkomen → ALTIJD `Veco · Beutech · Euromaster` (gefixeerde volgorde, één keer Beutech per blok).
2. **Engineering-doc-vocabulaire** — gebruik bewust: "spec-sheet", "doc-id", "metric-block", "manifest", "schematic", "byline".
3. **Tech-terminology mixin** — `stack`, `pipeline`, `throughput`, `recovery`, `reverse-engineered`. Niet over-gebruiken; max 1-2 per paragraaf.
4. **Numbers > adjectives** (audit-rule) — `47% response rate`, niet "veel response". `€519k pipeline recovery`, niet "groot project". `4 jaar bij Veco`, niet "lange tijd".
5. **Jij-vorm altijd** — niet U, niet wij.
6. **"Ik" niet "wij"** — autoriteit-positie, single-author voice.
7. **Klantnamen noemen** — `Veco · Beutech · Euromaster` (in die volgorde).
8. **Geen jargon** — geen PLC/SCADA/CMMS abbreviations; schrijf voor HR Directors, niet engineers.
9. **Specifieke prompts/cases**, geen abstracties.
10. **Polariseren mag, beledigen niet** — markt mag, mensen niet.

### 6. Forbidden patterns

**Visueel:**
- ❌ **Geen Aebi Schmidt-references** (campagne, post, asset, alt-text — niets)
- ❌ **Geen Fraunces / serif-fonts ANY** — alleen Inter + JetBrains Mono
- ❌ **Geen rounded corners > 4px** — corporate-tech feel, sharp edges, max 2-4px op CTA-buttons
- ❌ **Geen lichte modus / dark mode toggle** — site is single-mode (warm-grey + blue-deep contrast)
- ❌ **Geen casual / hoodie / generic stock visuals** — alleen real-shot lichtgrijs blazer + ronde bril
- ❌ Geen donker pak / navy blazer (verkeerde authority-signaal)
- ❌ Geen smiling-into-camera fake / AI avatars zonder foto-anchor
- ❌ Geen drop-shadows > 2px / glow-effects / gradients buiten cyan-orange-tape

**Inhoudelijk:**
- ❌ "Wij zijn een full-service partner"
- ❌ Generieke kandidaatpools spammen
- ❌ Werken met iedereen (alleen tech mid-market 50-800 FTE)
- ❌ Zonder data werken / abstract claims
- ❌ Afgezaagde HR-clichés ("passion for people", "synergy", "win-win")
